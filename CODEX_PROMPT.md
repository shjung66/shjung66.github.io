# Codex 작업 지시문

이 저장소는 교수 개인 홈페이지 겸 대학원 랩 홈페이지용 정적 사이트입니다. 사용자는 홈페이지 제작 경험이 없으므로, 코드는 단순하고 유지보수하기 쉽게 유지해야 합니다.

## 목표

일반적인 교수/랩 홈페이지처럼 다음 정보를 정리합니다.

- 연구자 프로필
- 연구 분야
- 현재 학생 및 학생연구원
- 졸업생
- 논문 및 보고서
- 프로젝트
- 강의 및 세미나
- 연락처

## 톤

- 과장된 홍보 문구를 쓰지 않습니다.
- “작은 연구공동체”, “함께 성장하는 커뮤니티” 같은 슬로건형 표현은 피합니다.
- 일반적인 대학 교수 홈페이지, 대학원 연구실 홈페이지와 유사한 담백한 문체를 사용합니다.
- 사실 중심으로 작성합니다.

## 기술 조건

- HTML, CSS, JavaScript만 사용합니다.
- 별도의 빌드 도구, 프레임워크, 데이터베이스를 추가하지 않습니다.
- GitHub Pages에서 바로 배포 가능해야 합니다.
- 모바일에서도 읽기 쉬운 반응형 레이아웃을 유지합니다.
- 접근성을 해치지 않도록 heading 구조, alt 텍스트, 링크 텍스트를 명확히 유지합니다.

## 수정 우선순위

1. `Your Name`, 소속, 이메일, 연구실 위치를 실제 정보로 교체합니다.
2. `profile.html`의 Biography, Education, Experience를 실제 이력으로 수정합니다.
3. `research.html`의 연구 분야를 실제 연구 분야로 수정합니다.
4. `people.html`에 현재 학생, 학생연구원, 졸업생 정보를 추가합니다.
5. `publications.html`에 논문, 발표, 보고서 목록을 추가합니다.
6. 필요하지 않은 메뉴는 삭제하거나 숨깁니다.
7. 사진 파일이 제공되면 `assets/img/`에 넣고 HTML에 연결합니다.

## 구성원 정보 처리 원칙

- 학생 이름, 사진, 이메일, 현재 소속은 공개 동의가 있는 경우에만 표시합니다.
- 공개 범위가 불명확한 정보는 표시하지 말고 비워둡니다.
- 졸업생의 현재 직장명이 공개 불가이면 `Current Field`만 표시합니다.

## 좋은 변경 예시

- `Research Area 1` → `Artificial Intelligence Applications`
- `Student Name` → `김민수`
- `Thesis Topic` → `제조 공정 데이터를 활용한 품질 예측 모델 연구`
- `Project Title` → `산업 데이터 기반 의사결정 지원 연구`

## 피해야 할 변경

- 불필요하게 화려한 애니메이션 추가
- 무거운 프레임워크 추가
- 문의 폼과 백엔드 서버를 갑자기 추가
- 동의 없는 개인정보 공개
- 마케팅 랜딩페이지 같은 문구 추가
